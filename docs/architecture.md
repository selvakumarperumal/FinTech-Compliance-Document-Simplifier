## Components

### Networking (VPC)

| Component | Description |
|-----------|-------------|
| VPC | 10.0.0.0/16 CIDR block |
| Public Subnets | 3 subnets across AZs (10.0.101-103.0/24) |
| Private Subnets | 3 subnets across AZs (10.0.1-3.0/24) |
| NAT Gateway | Single NAT for cost optimization |
| Internet Gateway | Public internet access |

### Compute (EKS)

| Component | Description |
|-----------|-------------|
| EKS Cluster | Kubernetes 1.28 |
| Node Group | t3.medium instances (2-4 nodes) |
| Pods | 2 replicas of the application |

### Storage & Registry

| Component | Description |
|-----------|-------------|
| ECR | Docker image repository |
| S3 | Terraform state storage |
| DynamoDB | Terraform state locking |

## Data Flow

1. **User Request**: User uploads document via HTTPS
2. **Load Balancer**: Routes request to Kubernetes service
3. **Kubernetes Service**: Forwards to available pod
4. **Application Pod**: 
   - Processes document
   - Calls NVIDIA LLM API
   - Generates simplified PDF
5. **Response**: PDF returned to user

## Security

- Private subnets for EKS worker nodes
- NAT Gateway for outbound internet access
- Security groups limiting ingress
- Encrypted S3 bucket for state
- Secrets managed via Kubernetes Secrets
